# Experience Page — Persona 3 Reload Social Link Inspired UI

The experience page should recreate the visual structure and energy of the Persona 3 Reload Social Link menu, but adapted for a personal portfolio experience section. The page should feel like a game menu: layered, sharp, blue, animated, asymmetrical, and highly stylized. It should not look like a regular resume page or a standard portfolio card layout.

The page should use the existing animated blue background from the website. The animated background should fill the entire screen behind the UI. There should be no additional character scene, environment image, hallway image, or other visual content behind the UI panels. The blue animated background itself is the only background layer.

The overall layout is divided into three main areas:

1. A large stylized title and experience information panel in the upper-left.
2. A bullet-point job description area on the left/middle-left.
3. A company identity area on the right, containing the company icon and technology list.

The entire page should feel like a layered collage made from sharp geometric panels, angled shapes, strong contrast, and oversized typography.

---

## 1. Background

Reuse the website’s existing animated blue background as the full-screen background for this page.

The background should remain visible behind all UI elements. It should provide the same deep blue, cyan, watery, animated, Persona-inspired atmosphere already used in the main menu.

No extra background image should be added behind the panels. Do not place a blurred office, school hallway, company photo, character scene, or any additional environment image behind the UI. The page should rely only on the animated blue background and the UI layers placed on top of it.

The UI panels should appear to float above this animated background.

---

## 2. Large Page Title: “EXPERIENCE”

At the top-left of the page, place a huge cropped title that says:

**EXPERIENCE**

This replaces the original “SOCIAL LINK” text.

The title should be:

* Very large
* White
* Bold
* Italic or slanted
* All uppercase
* Positioned near the top-left
* Partially cropped by the screen edges
* Used more as a graphic background element than a normal heading

The text should feel oversized and energetic. It should sit behind or partially behind the navigation controls and the main information panel.

It should not be centered. It should feel like it extends beyond the canvas, similar to the Persona 3 Reload menu title.

---

## 3. Previous / Next Navigation Prompts

The original LB / RB controller prompts should be replaced with portfolio navigation controls:

**Previous** and **Next**

These controls should sit near the top area, over or near the large “EXPERIENCE” title.

They should preserve the feel of game menu shoulder-button navigation, but with regular readable labels instead of controller buttons.

The navigation should include:

* A left-facing red arrow near “Previous”
* A right-facing red arrow near “Next”
* White or light text with a dark outline or strong contrast
* A slightly angled, game-like style

The purpose of these controls is to switch between different experiences.

They should feel like part of the Persona-inspired UI, not like normal website buttons. Avoid plain rectangular buttons. Use sharp edges, arrow shapes, bold text, and slight layering.

---

## 4. Main Experience Information Card

The upper-left section should contain a large black-and-white angled information card inspired by the Social Link card.

This card is the main identity panel for the selected experience.

It should be built from layered geometric shapes rather than a simple rectangle. It should include:

* A black header strip at the top
* A large white main panel underneath
* Sharp diagonal cuts
* Slanted edges
* A small angled label area on the left
* A large date range area on the right
* Strong black-and-white contrast

The card should look like a dynamic comic-book/game-menu banner.

---

### 4.1 Company Name Header

The black header strip at the top of the card should display the **company name**.

This replaces the original “Classmate” label.

Example values:

* McGill University
* Cineplex
* Befriend
* WatPlan
* University of Waterloo
* Company Name

The company name should be:

* White
* Italic or slightly slanted
* Centered or near-centered inside the black strip
* Medium-large
* Clear and readable

The black header should cut across the top of the white panel and should have angled edges.

---

### 4.2 Position / Location Angled Label

The small angled label on the left side of the card should display the **position and/or location of the experience**.

This replaces the original “ARCANA” label.

This label can contain concise text such as:

* Software Developer Intern
* Research Intern
* Montréal, QC
* Waterloo, ON
* Remote
* Internship
* Research

If both position and location are needed, they can be stacked or separated with a small divider.

The label should be:

* White text
* On a black angled shape
* Bold
* All caps or small caps
* Rotated or slanted slightly
* Visually attached to the left side of the main card

The goal is to make the position/location feel like a stamped UI tag.

---

### 4.3 Experience Title

The large main text inside the white panel should display the **experience title**.

This replaces the original “Magician” label.

Example values:

* Software Developer Intern
* Chemistry Research Intern
* Mobile App Founder
* Full-Stack Developer
* French Tutor
* AI Game Developer

The experience title should be the most important text inside the white panel.

It should be:

* Large
* Black
* Italic or stylized
* Highly readable
* Positioned toward the left/middle of the white panel
* More expressive than normal body text

It should feel like the main “name” of the experience.

---

### 4.4 Date Range

The original “RANK 1” section should become the **date range** of the experience.

Instead of showing “RANK” and a huge number, this section should show dates such as:

* May 2025 — Aug 2025
* Jan 2026 — Apr 2026
* Summer 2024
* 2023 — Present

The date range should be placed on the right side of the white card, where the large rank number originally appeared.

It should still feel bold and important, but it does not need to be as oversized as the original rank number.

The date range should be:

* Black
* Bold
* Large enough to be visually important
* Right-aligned or placed in the right portion of the card
* Integrated into the same white panel

The date area should preserve the feeling of the original rank block while making sense for a portfolio experience page.

---

### 4.5 Remove Star Row

The original Social Link star progression row should be removed.

Do not include the row of gray and black stars under the experience title.

The white information card should feel cleaner and more professional while still keeping the Persona-inspired shape language.

---

## 5. Left Job Description Area

Below the main experience information card, on the left or middle-left side of the page, include the job description.

This replaces the original narrative Social Link text area.

The job description should be written as resume-style bullet points, but the bullet symbols should be stars instead of regular dots.

The text should sit directly on top of the animated blue background, without a heavy rectangular container.

Each bullet point should:

* Start with a small star symbol
* Use white text
* Have a subtle shadow or glow for readability
* Be aligned in a clean vertical list
* Be spaced enough to remain readable
* Use a medium-large font size

Example structure:

★ Built and maintained a Flutter/Firebase mobile app with authentication, profile systems, image uploads, notifications, and real-time social features.

★ Designed Firestore data structures for friendships, user profiles, mutual friends, and proximity-based progression.

★ Improved user experience by implementing profile customization, infinite scrolling lists, and responsive mobile layouts.

The bullet points should feel like a stylized resume section inside a game menu. They should be readable and professional, but still visually connected to the Persona-inspired design.

Avoid placing these bullets inside a normal card unless readability requires a subtle transparent overlay. If an overlay is needed, it should be minimal, dark blue, semi-transparent, and angular.

---

## 6. No Central Character

The original screen contains a large central character silhouette behind the UI. This portfolio page should remove that character entirely.

Do not replace the central character with another person, avatar, anime figure, developer silhouette, laptop silhouette, or object.

The center area should remain visually open, allowing the animated blue background and UI panels to breathe.

The page should still feel dynamic through the angled panels, typography, company icon, and technology list rather than through a large character figure.

---

## 7. No Tarot Card

The original Social Link screen contains a floating tarot-style card near the center. This portfolio version should not directly recreate that tarot card.

Do not include a tarot card, arcana card, or floating symbolic card unless a later design decision adds a separate custom portfolio badge.

The page should keep the Social Link layout inspiration, but the portfolio version should be cleaner and less tied to the exact tarot concept.

---

## 8. Right-Side Company Icon Area

The right side of the page should replace the original character portrait with a large company icon or project icon.

This icon should act as the main visual identity for the selected experience.

The company icon should be:

* Large
* Positioned on the right side of the screen
* Partially cropped or placed dramatically, similar to how the original portrait fills the right side
* Mostly white, grayscale, or lightly blue-tinted when possible
* Integrated into the blue background with subtle transparency, glow, or shadow
* Not placed inside a plain square card unless the source logo requires containment

The icon should feel like a large visual cutout, not just a small logo.

For example:

* McGill could use a stylized university/lab-related icon
* Befriend could use a phone or social bubble icon
* WatPlan could use a roadmap/calendar icon
* A company could use its logo in a large monochrome treatment

The icon should not overpower the text, but it should provide strong visual identity on the right side.

---

## 9. Bottom-Right Identity / Technology Panel

At the bottom-right, include a dark navy angled information panel inspired by the original character nameplate.

This panel should contain:

1. The company or project name
2. A short one-sentence summary
3. A list of technologies used

The panel should be dark navy or deep blue, with angular edges and layered blue shapes behind it.

It should not be a plain rectangle. It should use sharp diagonal cuts and overlapping shapes to match the Persona-inspired UI.

---

### 9.1 Company or Project Name

At the top of the bottom-right panel, display the company or project name.

This replaces the original character name.

The name should be:

* Large
* Cyan
* Italic or slanted
* Highly visible
* Placed near the top-left of the panel

Example:

**Befriend**
**McGill University**
**WatPlan**
**Cineplex**

---

### 9.2 Short Experience Summary

Under the company or project name, include a short one-sentence summary of the experience.

This replaces the original character description.

The summary should be:

* White
* Bold or semi-bold
* Two to three lines maximum
* Clear and readable
* More compact than the bullet-point job description on the left

Example:

A mobile social app focused on proximity-based friendships, profile systems, image sharing, and real-time Firebase-backed interactions.

This summary should quickly explain what the experience is before the user reads the full bullet points.

---

### 9.3 Technology List

Inside the same bottom-right panel, include a list of technologies used during the experience.

The technologies can appear as small tags, chips, or compact text labels.

Examples:

* Flutter
* Firebase
* Firestore
* Cloud Functions
* Dart
* Git
* Python
* C++
* React
* Next.js
* MongoDB

The technology list should fit the Persona-inspired style. Avoid generic rounded pill tags if they look too modern/SaaS-like. Prefer sharper, angular tags or compact labels.

Technology tags should use:

* White or cyan text
* Dark blue or black backgrounds
* Thin cyan/white borders if needed
* Small spacing between tags
* Sharp or slightly slanted shapes

The list should be readable but visually secondary to the company/project name.

---

## 10. Back Button

The bottom-right should also include the website’s usual “Back to Menu” control.

This replaces the original small “B Back” controller prompt.

The back button should match the existing back-to-menu style already used elsewhere on the portfolio site.

It should remain small and secondary compared to the main experience content.

The button should return the user to the main menu.

---

## 11. Shape Language and Styling Rules

The entire page should follow these visual rules:

* Use sharp diagonal cuts instead of rounded rectangles.
* Use layered panels that overlap each other.
* Use strong black, white, navy, and cyan contrast.
* Use red only as a small accent for navigation arrows.
* Keep the layout asymmetrical.
* Let some text or panels feel slightly oversized or cropped.
* Avoid centered, clean, corporate portfolio layouts.
* Avoid plain cards unless they are heavily stylized.
* Avoid soft rounded SaaS-style UI components.
* Keep the design energetic, bold, and game-menu-like.

The page should feel like an interactive RPG menu adapted for a professional experience section.

---

## 12. Color Direction

The color palette should stay close to the original Persona 3 Reload Social Link screen:

* Bright cyan and electric blue for the animated background
* Deep navy for dark panels
* White for large title text and main readable text
* Black for strong contrast blocks
* Cyan for names, highlights, and accents
* Light gray only for secondary icon details if needed
* Red only for small previous/next arrows

The red should be used sparingly so it stands out.
